What changed

Built for the latest OpenClaw

This release focuses on the parts of the setup path that must stay dependable when OpenClaw changes its Gateway output, plugin registry and repair behavior.

01 / Gateway health

OpenClaw 7.1 status support

LocalClaw now reads the current Gateway health response and recognizes the new running state without reporting a false failure.

02 / Repair

Safer update recovery

The repair path reinstalls the Gateway service when needed, runs the supported doctor flow, restarts it and verifies health before continuing.

03 / Plugins

Cleaner plugin reconciliation

Official plugin metadata is refreshed while legacy or stale entries are handled conservatively to reduce post-update drift.

04 / Install state

No false "Ready" screen

If OpenClaw or the Gateway remains unhealthy, LocalClaw stops and reports the failure instead of presenting an installation as complete.
