LocalClaw 1.0.171

OpenClaw failures no longer end with a raw red log. LocalClaw now repairs common runtime problems automatically and gives every remaining error a clear, safe next action.

What changed

Automatic Gateway recovery

When Chat or Developer detects a stale or unreachable Gateway, LocalClaw refreshes the user service, verifies RPC health, and retries the message automatically.

Actionable error cards

Failures now explain what happened and offer the correct action. Technical output remains available in a collapsed, copyable details section.

Safe runtime repair

If OpenClaw package files are genuinely incomplete, LocalClaw creates a recovery point and reinstalls the same version before retrying. Configuration, projects, channels, and models are preserved.

A recovery path for every failure

Authentication, local model, session, timeout, Gateway, and unknown errors each lead to the relevant repair, Models screen, fresh session, retry, or redacted Help diagnostic.

Existing customers

Open Updates in LocalClaw and install version 1.0.171. Your OpenClaw configuration, channels, projects, and local models are preserved.
